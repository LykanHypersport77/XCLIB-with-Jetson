Hyperspectral Imaging System (HSIS)

Software Operations & Engineering Manual

This document details the configuration, deployment, and operation of the three core software modules driving the Hyperspectral Imaging System (HSIS). The software suite is split into headless flight operations, live ground testing, and offline spectral analysis.

Source Code & Version Control:
This software architecture is actively maintained and updated. For the latest code commits, bug fixes, or to contribute to the project, visit the official repository:

🔗 github.com/LykanHypersport77/XCLIB-with-Jetson
Target Hardware Environment:
  • Flight Computer: NVIDIA Jetson (ARM64) running Ubuntu / JetPack.
  • Camera System: Raptor Hawk (1944 x 1472 resolution, 12-bit native sensor depth).
  • Frame Grabber: EPIX PIXCI Frame Grabber utilizing XCLIB/XCAP libraries.
  • Ground Station: Windows 10/11 Lab Laptop (1080p display with DPI scaling handled automatically).

Module 1: Automated Flight Capture (auto_capture.py)

Environment: NVIDIA Jetson (Offline, In-Flight)

The Flight Capture script is designed as a headless, deeply embedded routine. Its sole purpose is to initialize the camera, catch 12-bit TIFF frames over the PCIe lane, and flush them directly to the storage drive as fast as possible. This process is strictly I/O bound.

Key Engineering Features

  • Zero External Dependencies: Built entirely using the Python Standard Library (ctypes, time, os). It requires absolutely no pip installations and runs natively on a fresh Jetson flash.
  • Headless Execution: Designed to be launched automatically by Ubuntu's systemd upon power-up, requiring no display or user interaction.
⚠️ CRITICAL THERMAL WARNING: Jetson Power Mode
Do not fly the Jetson in MAXN mode. MAXN powers all GPU cores and generates immense heat, which will cause a thermal shutdown inside a sealed reentry enclosure. Because the flight script utilizes 0% of the GPU, you must set the Jetson to 15W Mode (e.g., MODE_15W_6CORE). This allocates maximum clock speed to the CPU cores handling disk I/O while keeping thermals manageable.

Systemd Service Management

The capture script runs as a background daemon. To configure it for flight, you must manage it via the Ubuntu terminal.

1. To Enable & Start for Flight:

This command arms the payload. It tells the Jetson to launch the script automatically the moment it boots up.

sudo systemctl enable hsis_capture.service
sudo systemctl start hsis_capture.service

2. To Disable & Stop for Ground Testing:

Note: You must run these commands to stop the background capture loop before using the Ground Test GUI, otherwise the EPIX Frame Grabber will throw a "Device Busy" error!

sudo systemctl stop hsis_capture.service
sudo systemctl disable hsis_capture.service

3. To Monitor Service Health:

If you need to verify the script is successfully catching frames and saving TIFFs, view the live terminal output via the journal system:

sudo systemctl status hsis_capture.service
# Or to watch a live feed of the outputs:
sudo journalctl -u hsis_capture.service -f

Module 2: Ground Test GUI (ground_test.py)

Environment: NVIDIA Jetson (Bench Testing with Monitor) or Lab Laptop

The Ground Test script provides a live graphical interface to verify camera focus, pinhole alignment, and laser calibration before sealing the payload for flight.

Key Engineering Features

  • Real-Time Visualization: Captures and scales the raw 12-bit sensor data into an 8-bit viewable window using Tkinter and Pillow (PIL).
  • 15W Desktop Mode Compatibility: For GUI rendering on the Jetson, utilize the "15W Desktop" power profile to prevent UI stuttering, as it keeps more cores active for background display management.

Installation Requirements (Jetson)

Because this script uses UI libraries, it requires specific packages. If the Jetson is offline, bridge an internet connection via Ethernet from your laptop.

sudo apt update
sudo apt install python3-pip python3-tk
pip3 install Pillow numpy

Usage

  1. Connect a monitor, mouse, and keyboard to the Jetson.
  2. Execute the script: python3 ground_test.py.
  3. Use the Tkinter UI to manually trigger exposures, adjust lighting, and verify that the dispersion streaks are properly aligning with the sensor's physical pixels.

Module 3: Post-Flight Analysis (postanalysis.py)

Environment: Windows 10/11 Lab Laptop

The Post-Flight Analysis suite is the core data-processing engine. It extracts specific dispersion streaks from the raw flight TIFFs, corrects complex optical shear, and maps the integrated spectral data back into an interactive, spatial 2D representation of the pinhole array.

Key Engineering Features

  • Sub-Pixel Vector Sampling: Straightens optical dispersion shear by mathematically casting a centerline vector and sampling pixels perpendicularly, preventing division-by-zero errors on highly horizontal streaks.
  • Live Spaced Datacube Grid: Utilizes a decoupled "Two-Step Pipeline." Step 1 runs the core physics math to extract the 1:1 sensor data. Step 2 slices the optical data and layouts it into a padded, perfectly spaced 8x10 grid. Any blocked/shadowed pinholes render as pitch-black cells, visually recreating the physical shadow of the captured target.
  • Laser Auto-Calibration: A dedicated calibration tool runs a Savitzky-Golay peak detection filter across all dispersions to find the true laser dot while ignoring CMOS read noise. It instantly tags the dot with its calculated physical wavelength and colors the targeting crosshair using a dynamic Wavelength-to-RGB algorithm.
  • Dynamic Digital Bandpass: Auto-scaling, independent UI trackbars allow live cropping of the physical spatial slices, simulating a tunable bandpass filter.

Installation Requirements (Windows Laptop)

Open the Command Prompt and run:

winget install Python.Python.3.11
# Restart Command Prompt, then run:
pip install opencv-python numpy matplotlib scipy

Interactive UI & Usage

  1. Run the script: python postanalysis.py. A secure Windows file dialog will prompt you to select the raw `.tif` flight image.
  2. Main Interface (RAW View): The image loads with dynamic blue tracking lines overlaying the defined dispersion streaks.
  3. Spectrum Graphing: Left-Click near any blue tracking line. A Matplotlib window will spawn, plotting the raw 1D sensor data alongside the smoothed red curve for accurate SNR reading.
  4. Visual Datacube (R Key): Press 'R'. A massive, padded 8x10 grid will generate, showing the unwarped, 1:1 raw sensor slices mapped to their physical pinhole locations. Adjust the trackbars to slide a visual mask across the wavelengths live.
  5. Laser Peak Calibration (L Key): Press 'L'. A new window will overlay bright crosshairs directly onto the highest-intensity pixel of each dispersion, tagged with the true nanometer wavelength and logged to the terminal.
